1.  Данная работа развивает практические навыки по разработке web-сервисов для доступа к серверной базе данных. Теоретические сведения по созданию web-сервисов в среде NetBeans изложены в лекционном материале данного ЭРУД (лекция 12). Работа с серверной базой данных описана в лекции 7 настоящего ЭРУД.

 

2.  Прежде всего, следует создать web-проект.

3.  В него следует добавить web-службу. Для этого активизируем контекстное меню на имени проекта и выбираем п. Создать – Другое – Веб-Службы.

 

 

4.  Выбираем тип файла Веб-служба и жмем Далее. Вводим имя службы и имя пакета.

 

5.  Жмем Готово.  Система создает  заготовку сервиса.

 

6.  Добавляем в службу бизнес-методы для работы с базой данных:

 

-         Для соединения с базой данных

-         Для выполнения запроса на выборку

-         Для вставки новой записи

-         Для удаления записи.

7.   Для работы с базой данных нужно подключить к проекту архивный файл используемого драйвера (.jar).  Такой файл для MySql, Derby или ACCESS свой собственный. Для Derby он входит в состав дистрибутива GlassFish (два другие следует скачать в Интернет).

1.     Данная работа развивает практические навыки по разработке web-приложений для работы с серверными базами данных через персистентные классы. Теоретические сведения изложены в лекционном материале данного ЭРУД (лекция 16). Работа с серверной базой данных описана в лекции 7 настоящего ЭРУД.

2.     Структуру персистентного класса следует объявить в отдельном классе java.

3.     Персистентный класс должен быть «привязан к физической базе данных» через файлы конфигурации (map-файл):

 

<?xml version="1.0" encoding="UTF-8"?>

<!DOCTYPE hibernate-mapping PUBLIC "-//Hibernate/Hibernate Mapping DTD 3.0//EN"

"http://hibernate.sourceforge.net/hibernate-mapping-3.0.dtd">

<hibernate-mapping package="hiberlabe">

    <class name="stud" table="stud">

       <id name="fio" column="fio">

            <generator class="native"/>

        </id>

          <property name="age" column="age"/>

    </class>

</hibernate-mapping>

и файл hibernate.cfg.xml. Он стандартный и имеет такой вид:

<?xml version='1.0' encoding='utf-8'?>

<!DOCTYPE hibernate-configuration PUBLIC

    "-//Hibernate/Hibernate Configuration DTD//EN"

    "http://hibernate.sourceforge.net/hibernate-configuration-3.0.dtd">

<hibernate-configuration>

    <session-factory>

      <!-- Database connection settings -->

        <property name="connection.driver_class">org.apache.derby.jdbc.ClientDriver</property>

        <property name="connection.url">jdbc:derby://localhost:1527/MeineBase</property>

        <property name="connection.username">oleg</property>

        <property name="connection.password">german</property>

      <!-- SQL dialect -->

        <property name="dialect">org.hibernate.dialect.DerbyDialect</property>

      <!-- JDBC connection pool (use the built-in) -->

        <property name="connection.pool_size">1</property>

      <!-- Enable Hibernate's automatic session context management -->

        <property name="current_session_context_class">thread</property>

      <!-- Disable the second-level cache -->

        <property name="cache.provider_class">org.hibernate.cache.NoCacheProvider</property>

      <!-- Echo all executed SQL to stdout -->

        <property name="show_sql">true</property>

      <!-- Mapping files -->

        <mapping resource="stud.hbm.xml"/>

    </session-factory>

</hibernate-configuration>
